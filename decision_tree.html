Core Concept

의사결정 트리란 무엇인가요?

스무고개처럼 질문을 반복하여 데이터를 분류하는 직관적인 알고리즘입니다.

🌳 의사결정 트리(Decision Tree)란?

의사결정 트리는 한 번에 하나씩 질문을 하며 정답을 찾아가는 방식으로 데이터를 분류합니다. 마치 스무고개 게임처럼 "색깔이 노란색인가요?", "키가 320 이상인가요?" 같은 질문을 통해 데이터를 단계적으로 나누어 최종 답을 찾습니다.

💡 핵심 원칙: 맨 처음 질문(루트 노드)은 데이터를 가장 잘 구분하는 것으로 정해야 합니다! 불순도(Gini) 지수를 최소화하는 질문이 최적입니다.

✅ 장점

  • 직관적이고 시각적으로 이해하기 쉽다
  • 복잡한 문제도 과정이 명확하게 드러남
  • 수치형·범주형 데이터 모두 처리 가능

⚠️ 단점(과적합 주의!)

  • 과적합될 수 있음 (훈련 데이터에만 잘 맞음)
  • 트리 가지치기(Pruning)나 깊이 제한 필요
  • 데이터가 조금만 바뀌어도 트리가 크게 변할 수 있음
🌱
루트 노드

첫 번째 질문

데이터 전체를 가장 잘 나누는 질문. 불순도를 가장 많이 낮추는 기준이 선택됩니다.

🔀
내부 노드

추가 질문

분류된 그룹을 다시 나누는 질문. 각 그룹 안에서 가장 좋은 기준을 찾습니다.

🍃
리프 노드

최종 결과

더 이상 나눌 수 없거나 나눌 필요가 없는 최종 분류 결과입니다.

📉
Gini 불순도

분할 기준

데이터 집합이 얼마나 섞여있는지를 나타냄. 0에 가까울수록 순수한 분류입니다.

❓ 어떻게 작동하나요?

의사결정 트리는 X·Y 좌표값을 기준으로 데이터를 나누는 수직·수평선을 찾습니다. 색칠된 영역은 해당 클래스로 분류되는 구역입니다.

Parameters

⚙️ 파라미터가 트리에 미치는 영향

📏

최대 깊이
max_depth

트리가 얼마나 깊이 질문을 할 수 있는지 제한합니다.

작게: 단순한 트리, 과소적합 위험
크게: 복잡한 트리, 과적합 위험 ⚠️
🌿

리프 최소 샘플
min_samples_leaf

리프(최종 결과) 노드에 최소 몇 개의 데이터가 있어야 하는지 지정합니다.

작게: 더 세밀한 분류, 과적합 위험
크게: 더 단순한 분류, 안정적
✂️

가지치기
Pruning

과적합을 방지하기 위해 성능 향상에 도움이 되지 않는 가지를 제거합니다.

사전 가지치기: 학습 중 조기 중단
사후 가지치기: 학습 후 가지 제거
📊

불순도 기준
criterion

분할 기준으로 사용하는 지표입니다. 일반적으로 Gini 계수를 사용합니다.

Gini: 0(순수) ~ 1(불순) — 빠름
Entropy: 정보이득 기반 — 정확
Pros & Cons

⚖️ 의사결정 트리의 장점과 단점

✅ 장점

🎯 직관적·시각적

트리 구조를 그림으로 보여주므로 복잡한 문제도 그 과정이 명확하게 드러납니다.

📋 설명 가능

어떤 기준으로 분류했는지 규칙을 명확히 설명할 수 있어 해석이 쉽습니다.

🔧 전처리 최소화

데이터 정규화/표준화 없이도 작동하며 수치형·범주형 데이터 모두 처리 가능합니다.

❌ 단점

⚠️ 과적합(Overfitting)

트리가 훈련 데이터에 너무 맞추어지면 새로운 데이터에 잘 작동하지 않습니다. 가지치기나 깊이 제한이 필요합니다.

📊 데이터 변화에 민감

데이터가 조금만 바뀌어도 트리 구조가 크게 달라질 수 있습니다.

📏 직선 경계만 가능

X·Y 축에 평행한 선으로만 분류하므로 대각선 경계가 필요한 문제에서 성능이 떨어집니다.

🌲 과적합 방지 방법

  • 최대 깊이(max_depth) 제한
  • 리프 노드 최소 샘플 수 증가
  • 가지치기(Pruning) 적용
  • 앙상블(Random Forest) 사용

🚀 의사결정 트리의 발전형

  • 랜덤 포레스트: 여러 트리 결합
  • Gradient Boosting: 오류 순차 개선
  • XGBoost: 현재 가장 많이 사용

의사결정 트리에서 루트 노드에 어떤 특성을 배치해야 하는지 이유와 함께 설명하세요.

루트 노드에는 데이터 전체를 가장 잘 구분하는 특성을 배치해야 합니다.
기준: Gini 불순도를 가장 많이 감소시키는 특성이나 질문을 루트 노드로 선택합니다.
예시: 교과서 그림 II-80에서 색상이 루트 노드인 이유는, 색상을 기준으로 전체 데이터를 나누면 노란색(기린+호랑이), 회색(코끼리), 갈색(원숭이)으로 잘 나누어지기 때문입니다.
5

의사결정 트리가 스무고개와 유사한 점과 다른 점을 각각 설명하세요.

유사한 점: 한 번에 하나씩 예/아니오 질문을 하며 답을 좁혀나가는 방식이 동일합니다.
다른 점:
① 스무고개는 사람이 질문을 선택하지만, 의사결정 트리는 데이터를 가장 잘 나누는 질문을 자동으로 선택합니다.
② 스무고개는 최대 20번의 질문이 있지만, 의사결정 트리는 Gini 불순도를 기준으로 필요한 만큼만 질문합니다.
③ 의사결정 트리는 대량의 데이터에서 패턴을 학습할 수 있습니다.
