Classification

분류(Classification)란 무엇인가요?

정해진 범주에 따라 데이터를 나누는 것입니다. 로지스틱 회귀는 그중 이진 분류 문제를 해결합니다.

🔀 이진 분류(Binary Classification)란?

✅ / ❌

이진 분류의 특징

분류 결과가 딱 두 가지인 경우입니다. 결과를 0 또는 1로 표현합니다.

1 = 합격 / 스팸 / 양성 / 당뇨 있음
0 = 불합격 / 정상 / 음성 / 당뇨 없음
📧

실생활 이진 분류 예시

  • 📬 이메일 → 스팸(1) or 정상(0)
  • 📋 시험 → 합격(1) or 불합격(0)
  • 🏥 검사 → 양성(1) or 음성(0)
  • 🩸 혈당 → 당뇨(1) or 정상(0)

💡 왜 선형회귀로 분류하면 안 될까요?
선형회귀는 연속된 숫자(예: 62.5kg)를 예측하는 것에 특화되어 있습니다. 분류에 사용하면 결과가 0과 1 범위를 벗어나 -1이나 2같은 의미없는 값이 나올 수 있어요. 그래서 결과를 0~1 사이로 압축해주는 시그모이드 함수가 필요합니다!

Core Mechanism

로지스틱 회귀는 어떻게 작동할까요?

선형 계산 결과를 시그모이드 함수에 통과시켜 0~1 사이의 확률로 변환합니다.

📋
Step 1

분류 데이터 표시

합격/불합격처럼 결과가 1 또는 0으로 나뉜 데이터를 준비합니다.

🔢
Step 2

선형 식 계산 (z)

선형회귀처럼 가중치를 곱해 수치 z = ax + b를 계산합니다.

〜
Step 3

시그모이드 적용

무한한 z 값을 시그모이드 함수에 통과시켜 0~1 사이 확률로 압축합니다.

📍
Step 4

결정 경계 탐색

확률이 0.5(50%)가 되는 지점을 찾아 두 그룹을 나누는 경계선을 결정합니다.

💡 핵심 정리

  • 로지스틱 회귀는 이진 분류(0 or 1) 문제를 해결하는 알고리즘입니다.
  • 선형 계산 결과를 시그모이드 함수에 통과시켜 0~1 사이 확률로 만듭니다.
  • 확률 ≥ 0.5이면 클래스 1, 확률 < 0.5이면 클래스 0으로 분류합니다.

⚠️ 주의할 점

  • 이름에 '회귀'가 붙어 있지만 실제로는 분류 알고리즘입니다.
  • 선형 결정 경계만 그릴 수 있어, 복잡한 비선형 문제에는 한계가 있습니다.
Sigmoid Function

〜 시그모이드 함수란?

어떤 숫자든 0과 1 사이로 압축하는 S자 모양의 함수입니다.

시그모이드 함수 σ(z) = 1 / (1 + e−z)

σ(z) = 1 / (1 + e−z)

🔑 핵심 특성

  • z = 0일 때 → 출력값 0.5 (50%)
  • z가 매우 크면 → 출력값 1에 수렴 (100%)
  • z가 매우 작으면 → 출력값 0에 수렴 (0%)
  • 항상 0과 1 사이의 값만 출력

🏫 이해하기 쉬운 비유

자동문 센서를 생각해보세요! 사람이 멀리 있으면 문이 안 열리고(0), 가까이 있으면 열립니다(1). 시그모이드 함수는 "얼마나 가까운가"를 확률(0~1)로 부드럽게 변환해주는 것과 같습니다.

🎮 z값을 바꿔보세요!

−60+6
z = 0.0  →  σ(z) = 0.500  (50% → 판단 불가)

Limitations

⚠️ 로지스틱 회귀의 한계

로지스틱 회귀는 이진 분류에 유용하지만, 다음과 같은 상황에서는 한계가 있습니다.

📏

선형 경계의 한계

로지스틱 회귀는 직선(선형)으로만 두 클래스를 나눕니다. 데이터가 원형이나 나선형으로 분포되어 있다면 제대로 분류할 수 없습니다.

✗ 원형 분포 / ✗ 나선형 분포
→ SVM, 신경망 등 사용 권장
🔢

이진 분류만 가능

기본 로지스틱 회귀는 0과 1, 두 가지 분류만 가능합니다. 개/고양이/토끼처럼 3개 이상의 클래스를 분류하려면 다중 클래스 기법이 필요합니다.

✗ 3개 이상 클래스 분류
→ Softmax, 다중 클래스 분류 사용
📊

선형 관계 가정

독립 변수(x)와 결과(y) 사이에 선형 관계가 있다고 가정합니다. 비선형 관계를 가진 복잡한 데이터에서는 예측 성능이 크게 떨어집니다.

✗ 복잡한 비선형 패턴
→ 결정 트리, 신경망 사용 권장

💡 그럼 언제 로지스틱 회귀를 사용해야 할까요?

✅ 적합한 경우

  • 결과가 두 가지인 이진 분류
  • 빠르고 간단한 모델이 필요할 때
  • 결과를 확률로 해석해야 할 때
  • 데이터 양이 많지 않을 때

❌ 부적합한 경우

  • 3개 이상의 클래스 분류
  • 비선형적인 복잡한 패턴
  • 이미지·음성 같은 고차원 데이터
  • 변수간 복잡한 상호작용이 있을 때

📚 개념 이해 문제

1

로지스틱 회귀는 어떤 종류의 문제를 해결하는 알고리즘인가요?

✅ 정답: 이진 분류(Binary Classification) 문제

로지스틱 회귀는 결과가 두 가지(0 또는 1)인 이진 분류 문제를 해결합니다. 예: 합격/불합격, 스팸/정상, 양성/음성 등. 이름에 '회귀'가 있지만 실제로는 분류 알고리즘입니다.

2

시그모이드 함수의 출력 범위와 z=0일 때 출력값은 무엇인가요?

✅ 정답: 출력 범위 0~1, z=0일 때 0.5

시그모이드 함수 σ(z) = 1/(1+e⁻ᶻ)는 어떤 입력 z가 들어와도 항상 0과 1 사이의 값을 출력합니다. z=0이면 σ(0) = 1/(1+1) = 0.5입니다. 이 0.5가 분류 기준(임계값)이 됩니다.

3

로지스틱 회귀 모델이 출력한 값이 0.72라면, 이 데이터는 어느 클래스로 분류될까요?

✅ 정답: 클래스 1

출력값 0.72는 임계값 0.5보다 크므로 클래스 1로 분류합니다. 예를 들어 합격/불합격 문제라면 '합격(1)'으로 판정됩니다. 확률이 72%이므로 "이 학생이 합격할 확률은 72%입니다"라고 해석할 수 있습니다.

🔢 계산 문제

4

시그모이드 함수 σ(z) = 1/(1+e−z)에서 z = 2일 때 대략적인 출력값은 얼마인가요? (e ≈ 2.718)

✅ 정답: 약 0.88 (88%)

σ(2) = 1 / (1 + e⁻²)
e⁻² = 1/e² ≈ 1/7.389 ≈ 0.135
σ(2) = 1 / (1 + 0.135) ≈ 1 / 1.135 ≈ 0.880

z = 2이면 확률 88% → 클래스 1로 분류됩니다.

5

당의 섭취량(x)으로 당뇨병을 예측하는 모델의 공식이 z = 0.8×x − 4일 때, 당의 섭취량이 5g인 환자의 당뇨병 확률은 0.5보다 크다고 할 수 있을까요?

✅ 정답: 아니오 (0.5보다 작음)

z = 0.8 × 5 − 4 = 4.0 − 4 = 0
σ(0) = 1 / (1 + e⁰) = 1 / 2 = 0.5 (정확히 50%)

z = 0이므로 확률이 정확히 0.5입니다. 이 경우 "경계선상에 있다"고 하며, 보통 클래스 1(당뇨 있음)로 분류하거나 판단을 보류합니다.

⚠️ 한계 관련 문제

6

로지스틱 회귀로 강아지 / 고양이 / 토끼를 분류하려 합니다. 기본 로지스틱 회귀를 그대로 사용할 수 있을까요? 이유를 설명하세요.

✅ 정답: 사용할 수 없습니다.

기본 로지스틱 회귀는 결과가 2가지(0 또는 1)인 이진 분류만 지원합니다. 강아지/고양이/토끼처럼 3개 이상의 클래스를 분류하려면 Softmax 회귀(다항 로지스틱 회귀) 또는 다른 다중 클래스 분류 알고리즘을 사용해야 합니다.
